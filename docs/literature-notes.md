Content-based filtering
Content-based filtering is one of the most commonly used approaches in recommender systems. The idea behind it is straightforward: recommend items that are similar to items the user has already liked, based on their features or attributes. In the context of books, this means looking at metadata such as genre, author, and publication year. If a user has finished several Fantasy novels, the system identifies other books tagged with the Fantasy genre and recommends those. I implemented this approach in Bookish as SQL Variant A, where the algorithm extracts the genres from a user's reading history and scores unread books by how many of those genres they share. The main advantage of content-based filtering is that it is transparent and easy to explain — it is always clear why a particular book was recommended. The main limitation is that it tends to recommend more of the same, rarely suggesting anything outside the user's established preferences, and it depends entirely on the quality of the metadata available (Adomavicius and Tuzhilin, 2005).

Collaborative filtering
Collaborative filtering takes a different approach — instead of looking at the features of items, it looks at the behaviour of other users. The underlying assumption is that users who have agreed in the past tend to agree in the future, so if two users have rated the same books similarly, they probably share similar taste. The system then recommends books that one user enjoyed to the other. This is the approach used by platforms like Netflix and Spotify, where the scale of user data makes it extremely effective. The main advantage is that it can surface unexpected recommendations across genres that a content-based system would never find. However, it requires a large amount of user interaction data to work well, which is why I did not implement it in Bookish, at thesis scale, with a limited number of real users, there simply is not enough interaction data to compute meaningful user similarities (Adomavicius and Tuzhilin, 2005).

The cold-start problem
One of the most well-known challenges in recommender systems is the cold-start problem. It refers to the situation where a new user joins the platform with no history, no rated books, no reading list, nothing for the algorithm to work with. Without any data about the user, the system has no basis for personalisation and can only fall back on generic popular recommendations, which defeats the purpose of having a recommendation system at all. This problem affects content-based filtering too, but it is especially damaging for collaborative filtering, which needs data from both the new user and other similar users to function. To address this in Bookish, I designed an onboarding quiz that every new user completes before accessing the catalogue. The quiz asks them to select at least three favourite genres and to search for books they have already read. This gives the recommendation algorithm an immediate starting point, so that even on a user's first visit, the recommendations reflect their actual preferences rather than just what is globally popular (Schein et al., 2002).

# Platform Analysis
# Goodreads and StoryGraph: 0observed limitations


Goodreads

1. Recommendations have no explanations
When Goodreads recommends a book, it just shows
the cover and title with no explanation of why
that book was suggested. There is no mention of
which genre it matches or which book in my history
it relates to. This makes it difficult to judge
whether a recommendation is actually relevant to
my taste or just something that is popular with
everyone.

2. Recommendations are dominated by bestsellers
Almost every recommendation I received was a book
I had already heard of. The system seems to favour
books with the highest number of ratings overall
rather than books that match my specific reading
history. There is very little chance of discovering
lesser-known titles that might actually suit my
preferences better.

3. No solution for new users with no history
When I first created my account and added only
5 books, the recommendations felt almost identical
to what any other new user would see. The platform
did not ask me about my preferences upfront and
it took adding many more books before anything
started to feel personalised

4. The interface feels cluttered and outdated
The Goodreads homepage puts a social activity
feed, advertisements, reading challenges and
recommendations all on the same page. It is hard
to find the recommendations section without
scrolling past a lot of unrelated content. The
overall design looks like it has not been updated
in a long time.

5. Reading progress tracking is very basic
You can mark a book as currently reading and
update your page number manually, but there is
no visual progress bar and no summary of your
reading habits over time. The statistics side of
the platform is quite limited compared to what
a dedicated reading tracker could offer.

6. No transparency about how recommendations work
Goodreads uses a single recommendation approach
and gives users no information about how it works.
There is no way to understand why certain books
appear and others do not, which makes it hard to
trust or evaluate the suggestions.


 StoryGraph

1. Explanations are limited to mood tags
StoryGraph does better than Goodreads by showing
mood and pace tags on each recommendation, for
example fast-paced or tense. However it does not
provide a written explanation for why a specific
book was matched to me. I can see the tags but
not the actual reasoning, which still leaves me
uncertain about whether the book is a good fit.

2. Onboarding is better but still has problems
StoryGraph asks new users to rate a small set of
books during setup, which is an improvement over
Goodreads. However if a new user has not read many
of the suggested books, which is common for
younger readers, the setup process is hard to
complete properly and the first recommendations
still feel quite generic.

3. Some features require a paid subscription
Certain statistics and reading insights are only
available to premium subscribers. For a free user
the statistics page feels limited, which reduces
the value of the platform for casual readers who
do not want to pay a monthly fee.

4. The catalogue has gaps
Several books I tried to add were not in the
StoryGraph database, or had incomplete information
such as missing genre tags, no cover image or an
incorrect author name. This is especially noticeable
for books originally published in other languages
or for less mainstream titles.

5. The interface takes time to figure out
While StoryGraph looks cleaner than Goodreads,
the difference between the various lists such as
want to read, currently reading and did not finish
is not immediately obvious to a new user. I found
myself adding books to the wrong list and having
to work out how to move them, which was frustrating
at first.

6. No AI or explainable recommendations
Like Goodreads, StoryGraph uses an algorithmic
approach with no written justification for
individual recommendations. There is no way for
a user to ask why a book was suggested or to get
a readable explanation of why it was matched to
their profile.



 The Gap That Bookish Addresses

Neither platform provides written explanations
for individual book recommendations. Both have
problems with new users who have no reading
history yet. Neither platform lets users see and
compare two different recommendation approaches
at the same time.

Bookish tries to address all three of these gaps.
The onboarding quiz collects genre preferences
and prior reading history before the user even
reaches the catalogue, which solves the new user
problem. The Gemini integration generates a
written explanation for each AI recommendation
so users can understand why a book was suggested.
The home feed shows both the SQL based and the
AI generated recommendations side by side, making
it easy to compare the two approaches directly.
